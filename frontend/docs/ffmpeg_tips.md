# 🎬 FFmpeg.wasm 비디오 합성 트러블슈팅 가이드

이 문서는 브라우저 기반 비디오 합성 시 발생할 수 있는 주요 문제들과 해결 방법을 기록합니다.

## 1. 영상 재생 시간 0초 문제 (0s Video Duration)

### 증상
합성된 MP4 파일이 다운로드되지만, 재생 시 시간이 0초로 표시되거나 재생이 되지 않음.

### 원인
1.  **`shortest=1` 옵션의 오작동**: `overlay` 필터 등에 사용되는 `shortest=1`은 입력 스트림 중 가장 짧은 것에 맞춰 출력을 종료합니다. `MediaRecorder`로 녹화된 WebM 파일은 메타데이터상 재생 시간이 0이거나 매우 짧게 인식될 때가 많아, 이 옵션이 있으면 전체 출력을 0초로 잘라버립니다.
2.  **WebM 메타데이터 부재**: 브라우저 녹화본은 파일 끝에 메타데이터가 붙거나 누락된 경우가 많아 FFmpeg이 타임스탬프를 제대로 읽지 못합니다.
3.  **MP4 컨테이너 구조**: `moov` 아톰이 파일 끝에 있으면 일부 플레이어에서 즉시 재생 시간을 계산하지 못합니다.

### 해결책
-   **`shortest=1` 제거**: 배경(base)의 길이를 `-t` 또는 `d=4` 등으로 명시하고, 오버레이 시에는 이 옵션을 제거하여 입력 영상의 메타데이터 오류에 영향을 받지 않도록 합니다.
-   **`-fflags +genpts`**: 입력 파일의 누락된 타임스탬프를 강제로 생성합니다.
-   **`-movflags +faststart`**: MP4 메타데이터를 파일 앞부분으로 옮겨 스트리밍 및 재생 호환성을 극대화합니다.
-   **`-avoid_negative_ts make_zero`**: 타임스탬프가 음수로 시작하는 오류를 방지합니다.

## 2. 메모리 부족 및 충돌 (OOM / Worker Crash)

### 증상
합성 중 브라우저 탭이 새로고침되거나 "FFmpeg failed" 코드가 발생함.

### 해결책
-   **`-threads` 제한**: `ffmpeg.wasm`은 멀티스레딩 지원이 불안정할 때가 있으므로 `-threads 2` 정도로 낮게 설정합니다.
-   **`-preset ultrafast`**: 인코딩 속도를 높이고 CPU 점유를 줄입니다.
-   **`-crf` 조절**: 높은 숫자(예: `32`~`38`)를 사용하여 파일 크기를 줄이고 메모리 사용량을 낮춥니다.
